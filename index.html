¿Qué es una API?

Una guía completa, clara y moderna

1. ¿Qué es una API?

API significa Application Programming Interface (Interfaz de Programación de Aplicaciones).

En palabras simples: es un puente que permite que dos programas diferentes se comuniquen entre sí sin necesidad de conocer cómo funciona el otro internamente.

Analogía del restaurante:

• Tú eres el cliente (tu aplicación)
• El mesero es la API
• La cocina es el servidor o base de datos

Tú no entras a la cocina. Solo miras el menú, pides y recibes tu plato listo.

2. ¿Para qué sirven las APIs?

Sirven para conectar aplicaciones y reutilizar funcionalidades sin reinventar la rueda.

App del clima
Obtiene datos meteorológicos en tiempo real.
Iniciar sesión con Google
Conecta tu app con cuentas de Google de forma segura.
Google Maps
Muestra mapas y calcula rutas fácilmente.
Publicar en Instagram
Automatiza publicaciones y marketing.

3. Cómo funciona una API (Paso a paso)

  1. Haces una petición (Request) mediante HTTP.
  2. La API valida tu acceso (API Key o token).
  3. La API procesa la solicitud internamente.
  4. Recibes una respuesta, normalmente en formato JSON.
{
  "ciudad": "Madrid",
  "temperatura": 18,
  "descripcion": "Parcialmente nublado",
  "humedad": 65
}

4. Tipos principales de APIs

  • REST – La más usada (usa URLs y métodos GET, POST, PUT, DELETE).
  • GraphQL – Permite pedir exactamente los datos que necesitas.
  • SOAP – Más antigua, usada en empresas y bancos.
  • APIs locales – Bibliotecas como Pandas en Python.

5. Ejemplo práctico en Python

import requests

respuesta = requests.get(
    "https://api.openweathermap.org/data/2.5/weather",
    params={"q": "Madrid", "appid": "TU_CLAVE", "units": "metric"}
)

datos = respuesta.json()
print(f"Temperatura en Madrid: {datos['main']['temp']}°C")

6. Ventajas e Inconvenientes

Ventajas: Ahorro de tiempo, escalabilidad, seguridad y reutilización.

Inconvenientes: Dependencia externa, posibles costos y riesgos de seguridad si no proteges tus claves.
